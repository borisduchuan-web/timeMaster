# 一、产品定位

本产品是一个由 AI 统筹任务、能力、时间与成长的任务履约平台。

平台核心目标不是做传统抽佣型接单软件，而是建立一个“人能做什么、市场需要什么、AI 如何组织任务”的劳动能力网络。

一句话定义：

通过 AI 理解每个人的真实能力，把市场需求拆成可执行任务，并按照等级、声望、技能熟练度公平分发，让普通人和失业手艺人靠完成任务获得收入和成长。

---

# 二、核心差异

## 1. 平台不抽成

平台不从用户单笔劳动收入中抽取佣金。

用户完成任务后，任务约定劳动报酬应完整进入用户账户。

平台收入与用户劳动报酬分账：

* 用户收入：来自任务劳动报酬
* 平台收入：来自市场项目统筹费、企业服务费、政府/公益项目服务费、AI 工具增值服务、组织效率收益
* 禁止机制：不得通过隐藏比例、提现手续费、强制会员等方式变相扣减用户劳动收入

## 2. 平台主动统筹市场需求

平台不是只让用户自由抢单。

平台需要主动从市场侧获得需求，并由 AI 拆解为标准化任务：

* 企业任务
* 本地生活任务
* 社区服务任务
* 公益协作任务
* 数据整理任务
* 技能型履约任务
* 长期项目的子任务

AI 根据用户劳动能力、当前时间、地理位置、等级、声望和技能熟练度进行匹配。

## 3. 任务是唯一成长入口

用户不能靠充值、购买会员、刷资料直接提高等级。

等级、声望、技能熟练度必须来自真实任务行为：

* 完成任务
* 通过验收
* 按时履约
* 获得正向评价
* 完成公益任务
* 通过平台技能验证任务

学习课程可以帮助用户获得“练习建议”或“新手任务推荐”，但不能直接提升正式等级。

# 四、产品原则

## 1. 劳动优先

平台尊重真实劳动，不用流量、打赏或抽佣逻辑压过劳动价值。

## 2. 成长公平

用户的成长来自真实任务记录，而不是付费能力。

## 3. 新人可进入

平台必须设计新手任务和公益任务，让没有历史记录的人也能开始积累等级、声望和技能熟练度。

## 4. AI 辅助，不替代最终责任

AI 可以推荐、匹配、提醒、审核材料和识别风险，但关键处罚、封禁、争议裁决必须有人类复核。

## 5. 数据归用户

个人技能、履约记录、作品案例、知识库和成长档案属于用户的重要资产，用户应能查看、导出、授权和撤回授权。

```text

→ AI 初验 + 人工/需求方验收
→ 用户获得报酬
→ 等级、声望、技能熟练度更新
→ 个人知识库沉淀
→ 后续获得更匹配、更高价值任务
```

# 七、功能模块设计

# 7.1 AI 及用户通讯模块

## 模块目标

AI 既是用户的任务助理，也是平台的调度解释器、履约教练和成长顾问。

用户与 AI、用户与用户、用户与平台之间的通讯都应围绕任务履约展开。

## 功能清单

### 1. AI 新手引导

功能：

* 通过对话采集用户基本情况
* 识别用户已有技能
* 询问可工作时间、地点、工具、经验
* 推荐适合的新手任务
* 生成初始技能画像

示例问题：

* 你以前做过哪些工作？
* 你现在每天大概什么时间方便接任务？
* 你有哪些工具或设备？
* 你能接受线上任务、线下任务，还是都可以？

### 2. AI 任务讲解

用户进入任务详情页后，AI 需要把任务翻译成用户能理解的执行说明。

功能：

* 任务目标解释
* 操作步骤拆解
* 风险提醒
* 所需工具提醒
* 交付标准说明
* 预计耗时说明
* 是否适合当前用户的原因说明

### 3. AI 履约助手

用户执行任务过程中，AI 提供实时帮助。

功能：

* 步骤提醒
* 现场问题问答
* 图片/语音识别
* 材料检查
* 交付物预审
* 异常情况上报

### 4. 用户通讯

通讯对象：

* 服务者与需求方
* 服务者与平台客服
* 服务者与 AI 助手
* 项目组成员之间

功能：

* 文字消息
* 语音消息
* 图片/视频
* 位置共享
* 任务节点通知
* 系统模板回复
* AI 自动摘要
* 敏感信息保护

### 5. AI 沟通保护

平台需要保护弱势劳动者，避免用户在沟通中被诱导私下交易、压价、辱骂或欺诈。

风控能力：

* 识别私下交易引导
* 识别辱骂和威胁
* 识别虚假需求
* 识别超出任务范围的额外要求
* 识别压价行为
* 生成争议摘要

# 7.2 任务接发模块

## 模块目标

把市场需求变成可分发、可执行、可验收、可结算的标准任务。

## 任务来源

* 平台自营市场项目
* 企业需求
* 政府/社区项目
* 公益组织任务
* 个人用户需求
* 平台训练任务
* 技能验证任务

## 任务类型

### 1. 新手任务

给无等级、低声望、低熟练度用户使用。

特点：

* 风险低
* 步骤清晰
* 交付标准明确
* 收益较低或中等
* 可快速完成
* 用于建立初始履约记录

示例：

* 线上资料整理
* 简单图片标注
* 社区信息采集
* 店铺基础信息核验
* 低风险跑腿
* 师傅跟单辅助

### 2. 公益任务

用于平台冷启动、新人成长、社会价值建设。

特点：

* 可以提升声望
* 可以获得少量经验
* 可作为新用户初始信用证明
* 可由公益组织、社区、平台共同发布

示例：

* 社区老人手机使用辅导
* 公益物资整理
* 无障碍设施巡检
* 公共信息纠错
* 公益活动志愿协助

### 3. 技能任务

需要明确技能门槛。

示例：

* 家电维修
* 水电安装
* 家具修复
* 裁缝修改
* 摄影剪辑
* 电脑维护
* 文案整理

### 4. 项目拆解任务

平台把一个大项目拆成多个任务包。

示例：

* 企业门店巡检项目
* 城市社区服务项目
* 数据采集项目
* 本地商户数字化项目

### 5. 紧急任务

对响应速度要求高。

示例：

* 临时维修
* 紧急配送
* 活动现场支援
* 设备故障处理

## 任务生命周期

```text
草稿
→ 待审核
→ 待拆解
→ 待派发
→ 待接单
→ 已接单
→ 执行中
→ 待验收
→ 已完成
→ 已结算
→ 已归档
```

异常状态：

* 已取消
* 已超时
* 争议中
* 风控冻结
* 重新派发

## 接单规则

任务不是完全自由抢单。

平台采用“准入 + 推荐 + 保护”的接单机制。

准入条件：

* 等级满足
* 技能熟练度满足
* 声望满足
* 时间可用
* 地点可达
* 当前任务负载未超限
* 无相关风控限制

推荐规则：

* 优先推荐最适合用户的任务
* 新人保留一定的新手任务池
* 高声望用户获得更高价值任务优先权
* 长期未获得任务但表现稳定的用户获得公平曝光补偿

保护规则：

* 新人不能接高风险任务
* 低声望用户不能接高金额任务
* 超负荷用户不能继续接单
* 有争议未处理任务的用户限制部分接单

# 7.3 成长体系模块

成长体系由四个核心对象组成：

* 等级
* 声望
* 技能熟练度
* 个人知识库

## 1. 用户等级

用户等级代表用户在平台中的综合履约成长。

等级建议：

| 等级 | 名称 | 说明 | 可接任务 |
|---|---|---|---|
| Lv0 | 未验证 | 刚注册，尚无任务记录 | 仅可接引导/公益/训练任务 |
| Lv1 | 新手 | 完成基础任务，具备初始信用 | 低风险、低复杂度任务 |
| Lv2 | 稳定者 | 多次按时完成任务 | 常规任务 |
| Lv3 | 熟练者 | 某些技能稳定可靠 | 中等复杂度任务 |
| Lv4 | 专业者 | 具备专业技能和高履约质量 | 高价值技能任务 |
| Lv5 | 师傅/专家 | 可独立处理复杂任务 | 高难度任务、带新人任务 |
| Lv6 | 项目骨干 | 可承担协作和项目子管理 | 项目组长、质检任务 |

升级条件：

* 完成任务数量
* 任务通过率
* 按时率
* 争议率
* 声望分
* 技能熟练度
* 公益任务贡献

降级条件：

* 严重违规
* 多次超时
* 多次低质量交付
* 欺诈或虚假履约
* 高投诉率

## 2. 声望系统

声望代表用户是否值得信任。

声望建议采用 0-1000 分。

初始分：

* 未实名：300
* 已实名：400
* 完成新手任务：450-500
* 完成公益任务并通过验收：额外增加

加分行为：

| 行为 | 分值建议 |
|---|---|
| 按时完成任务 | +2 到 +10 |
| 获得好评 | +2 到 +8 |
| 连续完成无投诉 | +5 到 +20 |
| 完成公益任务 | +3 到 +15 |
| 主动上报异常并协助解决 | +5 到 +20 |
| 带新人完成任务 | +5 到 +20 |

扣分行为：

| 行为 | 分值建议 |
|---|---|
| 无故取消 | -5 到 -30 |
| 超时未说明 | -5 到 -20 |
| 交付不合格 | -10 到 -50 |
| 被有效投诉 | -20 到 -100 |
| 私下交易 | -50 到 -200 |
| 欺诈履约 | 直接冻结或封禁 |

声望等级：

| 声望分 | 等级 | 说明 |
|---|---|---|
| 0-299 | 风险 | 限制接单 |
| 300-499 | 初始 | 仅低风险任务 |
| 500-699 | 可信 | 常规任务 |
| 700-849 | 优质 | 优先派单 |
| 850-1000 | 卓越 | 高价值任务、项目任务 |

## 3. 技能熟练度

技能熟练度是按技能单独计算的，不等于用户总等级。

例如：

* 电器维修 Lv4
* 家具安装 Lv2
* 文案整理 Lv1
* 老人陪诊 Lv3

熟练度维度：

* 任务数量
* 任务难度
* 交付质量
* AI 过程评分
* 需求方评价
* 质检结果
* 证书/作品佐证
* 复购率

熟练度等级：

| 熟练度 | 名称 | 说明 |
|---|---|---|
| S0 | 未验证 | 仅用户自述 |
| S1 | 入门 | 完成低风险任务 |
| S2 | 基础 | 可独立完成标准任务 |
| S3 | 熟练 | 可处理复杂情况 |
| S4 | 专业 | 高质量稳定履约 |
| S5 | 专家 | 可质检、带教、参与标准制定 |

规则：

* 每个技能独立成长
* 不能用一个技能等级接另一个技能的高风险任务
* 用户可通过技能验证任务获得更快确认
* 长期不使用的技能进入“待复核”状态

## 4. 个人知识库

个人知识库是用户的长期能力资产。

内容来源：

* 用户自填经历
* AI 访谈记录
* 已完成任务
* 交付物
* 作品案例
* 证书
* 客户评价
* 常用工具
* 操作 SOP
* 失败复盘

用途：

* AI 更准确匹配任务
* 生成个人技能档案
* 生成接单简历
* 帮用户复盘成长路径
* 为高价值任务提供能力证明
* 支持未来跨平台技能护照

知识库权限：

* 默认私有
* 用户可选择对平台调度开放
* 用户可选择对需求方展示部分内容
* 用户可导出个人能力档案

# 八、AI 调度规则

## 1. 任务准入模型

任务先判断用户是否有资格接。

准入条件：

```text
can_accept =
level >= task.min_level
AND reputation >= task.min_reputation
AND skill_level >= task.min_skill_level
AND user_available = true
AND workload < max_workload
AND risk_status = normal
```

## 2. 匹配评分模型

通过准入后，系统计算匹配分。

```text
match_score =
skill_fit * 0.30
+ reputation_fit * 0.20
+ level_fit * 0.15
+ time_fit * 0.15
+ location_fit * 0.10
+ workload_balance * 0.05
+ growth_opportunity * 0.05
```

说明：

* skill_fit：技能是否真正匹配
* reputation_fit：声望是否可靠
* level_fit：等级是否合适
* time_fit：当前是否能完成
* location_fit：距离和到达时间
* workload_balance：避免任务集中到少数人
* growth_opportunity：给新人和成长用户合理机会

## 3. 新人保护派单

平台需要给新人设置独立任务池。

新人任务池规则：

* 每日保留一定比例任务给 Lv0-Lv1 用户
* 公益任务优先开放给新用户
* 低风险任务允许多人候选
* 新人连续完成任务后逐步进入普通任务池
* 新人被拒单时，AI 必须给出下一步成长建议

## 4. 高风险任务限制

以下任务必须提高准入门槛：

* 入户服务
* 高金额维修
* 老人儿童相关服务
* 夜间任务
* 涉及贵重物品
* 涉及安全责任
* 需要专业资质的任务

# 十三、运营机制

## 1. 新人启动

用户注册后必须有任务可做。

机制：

* AI 访谈生成初始画像
* 推荐 3-5 个新手任务
* 推荐 1-2 个公益任务
* 完成首单后展示成长变化
* 7 天内持续给出成长路径

## 2. 失业手艺人专项

平台应为手艺人设计快速建档路径。

机制：

* 上传过往作品
* 上传工具照片
* AI 访谈工作经历
* 老客户证明
* 技能验证任务
* 低风险试单
* 师傅等级认证

## 3. 公益任务体系

公益任务不是免费劳动的包装，必须有明确边界。

规则：

* 公益任务应标明是否有补贴
* 公益任务不能替代应付费的商业任务
* 公益任务主要用于社会价值、新人成长和社区协作
* 公益贡献可增加声望，但不能无限刷分

## 4. 平台不抽成的信任表达

在钱包和任务详情页明确展示：

* 任务报酬
* 用户到账
* 平台抽成：0
* 可能存在的第三方支付通道成本
* 平台收入来源说明
